Engine 2

A 3D Rendering engine to draw massive amounts of geometry efficiently using per cluster culling.

Meshes are divided into clusters, A compute shader determines currently visible clusters to minimize the amount of geometry drawn. Currently working to make this culling be done using heirarchical z.
Visibility buffer based rendering is used to save on GPU bandwidth. All triangles are loaded directly form storage buffers and all vertex attributes are software-interpolated manually in the pixel shader.
There is a complete seperation of Geometry, Material and Lighting passes.

Also includes an automatic asset management system for loading/unloading from disk.
A custom implementation of a Sparse Array Entity Component System (ECS).

C++17 | OpenGL

Vulkan Forward Clustered Renderer

Written from the ground up to render 3D GLTF Scenes.

Forward++ aka Forward Clustered Rendering achieved using a compute shader to determine which lights influence which chunk of the view frustum, followed by traditional forward rendering, but calculating lighting only for lights in the same chunk of the view frustum as the surfel.
